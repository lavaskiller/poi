# Archived documentation

This directory preserves dated plans, reviews, and specifications from the prototype's development. They are historical context, **not** the current product contract.

For current behavior, use:

- [API contract](../API.md)
- [Functional specification](../functional-spec.md)
- [Project README](../../README.md)

Historical documents may refer to old repository paths, unsupported UI pages, or planned functionality that was never implemented. Superseded reports and generated report artifacts are intentionally deleted from the working tree; recover a prior committed version from Git when needed.

The active activity record is in [`docs/reports/daily`](../reports/daily/README.md).
